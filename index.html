Variables and Strings

In this section we learned a little bit more of the power of Python building off the grammar information we learned in 2.1. We were introduced to variables in Python, which are used to create a name and store, or assign, a value to that name. To assign a value in Python we use the " = " symbol between the name of the variable and what value we want to store to that name.

For example if we want to store 28 to our variable "age", we would set it up as the follows: age (name of variable) = (the assignment) 28 (value). This is storing the value 28 to "age" which we can alter by adding or removing such as: new_age = age - 1. This would take the current value of age, which is 28, and subtract 1 to it thus creating and storing 27 to our new variable called "new_age".

The running topic is that computers are stupid and will output what we, the human, inputs. So in Python, 2 is a number while "2" is a string. So if we ran a code of 2 + 2, Python will give us 4. Whereas a code of "2" + "2" will give us "22". So we have to be very careful of what we write in our code.

So remember, variables can help in the following ways:

  • Improve code readability for users
  • Offer a valuable way to store data
  • Allow us a way to quickly change value, for example the "age" & "new_age" as stated above.

This lesson brought a lot of new information that will be used a lot during our time learning programming, not only in Python but in future languages and code itself! We learned that Boolean values can either be True or False and were introduced to If and While Loops. The If loop takes a Test Expression and if the statement is True, then the block of code will execute. If not, it will jump to the next block of code which we can execute with the line of code else. Below is an example of a very basic If loop:



def larger_number(a,b):
if number a > b
 return a
 else:
 return b

The example above will return the value a if it was bigger than b. Otherwise, or else, it will return b if the value b was bigger than a. It's very important to note that we should remember to use semicolon : when writing our code as errors will come up if not entered at the appropriate locations.

The While loop can keep running the same block of code while the test expression is true. Once the test expression is false it jumps to the next block of code and runs that. An example of the While loop is as follows:


def less_than_ten(a):
i = 1
while i less than 10
print i

This simple function will take whatever value you input to less_than_ten and while i is less than 10 it'll return the values. We can include the keyword break to jump out of a line of code but it's better practice not to include it unless we need to. Better to stick with the else method.

For the final two sections of Stage 2 we came across new datatypes and some tips to solving problems for the future. The new data types that we came across were: Lists and For Loops. On a personal note, loops have been an issue for me since I was introduced to them when I was learning about Ruby. With that being said, Lists have been an easier transition with Python as Lists, to me, are like Arrays that I worked with in Ruby. Lists can have anything inside them, whether it's characters, strings, numbers, or even other lists!.

Below is a very basic way to create a list.


list_example(this will be the name you give your list) = ['f', 'b', 'c', 'z']
print list

As we see above the list, list_example contains the following elements: 'f', 'b', 'c', 'z'. For lists we can include strings AND numbers. We can also call upon elements within the list as so: list_example[0]. This will call upon the first value, or as we know that [0] is appropriate for calling the first value in Programming. With lists we can also put lists within lists and call index for certain elements. For example:


example_two = [['a', 'b', 'c', 1],['dog','cat','fish','toy']]
print example_two[1][3]

The example_two will call the second list [1] and the then select the 4th element [3] and will return the value toy to us!

There are two main differences between strings and lists and they are:

  1. Lists can hold anything within them
  2. Lists support Mutation

Mutation allows us to change the value of a list even after we create it. For list_example we can change the 3rd element by calling the list list_example[2]='d' which will change, or mutate, the 3rd element thus giving us the updated list list_example = ['a', 'b', 'd', 'z']. It's also good to know that numbers CAN NOT be mutated. If we assign another variable to our list_example then we have two ways to call upon the same list, otherwise known as aliasing. When we have aliasing then we can change the element within the list and it'll change, ormutate, both of the lists! These changes will continue to occur until we assign one of the lists to another list.

With append we can add lists to lists and increase to them. It would be done but adding the .append to the list as so:


list_example.append(example_two)
print list_example

After appending example_two to list_example, list_example will become list_example = ['f', 'b', 'd', 'z' ['a', 'b', 'c', 1],['dog','cat','fish','toy']]. With the len operator, known as length, we can find out the length of lists and strings. We can find out the length of list_example with the len(list_example) and find out that there's only 3 elements. That's because we had appended the list example_two to list_example so being that example_two was a list itself, the operator len counts that as one whole element and not multiple ones within it.

The For Loop allows us to loop through elements within a list. The setup and result in this example is as following:


numbers = [1, 2, 3]
for e in numbers:
print e

As we see above, the for e in numbers: or for the elements in numbers print e or the elements. This will give the results as:


1
2
3

We can also directly check and see if values are in lists with the in opeartor and where values are in with the index operator. 2 in numbers will return True and numbers.index(2) will tell where the value is located, which in this case is 1 since we beginning counting at 0.
